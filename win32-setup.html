Win32 API tutorial

part 0: introduction

so, here we are. at this point, you should have confirmed that you want to learn how to use the Win32 API.

recommended IDEs

if you are confused on what IDE to use, here are the ones I recommend:

Visual Studio Community

this is a contraversial one, honestly it's not the best IDE for small scale projects, but I'm using it because of the MSVC++ compiler.
(in fact, i usually just use VSCode and just run it from Visual Studio code tools)
this is a good choice for first time Win32 developers as it does set up a lot of the stuff for you, so you can rest assured that your compiler options will work. takeaways: automatic setup, good debugging tools, but the tools are very complex and not very flexible.

Visual Studio Code (VSCode)

this is a common one. very similar to visual studio, but you have to set up a lot of the stuff you want to use.
less debugging tools, but so much more user-customisable. a lot less lightweight too!

Code::Blocks

also very lightweight, and it does set up a lot of things for you.
however, it is a lot less flexible at Visual Studio Code, and also doesn't support MSVC++ (Microsoft's C++ compiler)

Notepad/Notepad++

i've grouped these two together because notepad++ is slightly more optimised for coding.
they're both fine to use, but NO debugging tools at all so get used to the command line tools.

vim or nano

these are both command line editors and they're both good, but like notepad and notepad++, no debugging tools so have fun specifying every compiler option!

troubleshooting

if you've gotten to this point this code probably doesn't work.
try these steps to get your code to work.


Error LNKxxxx - undefined reference to WinMain@16 (or something like that)


if this happens, it means that it is trying to compile as a console application, and a Win32 GUI application is NOT a console application.
so if this happens, try these steps:

using MSVC++ (Visual Studio):
right click your project, click properties.
once on the properties page, on the left hand side, click Linker > SubSystem.
change the value to Windows (SUBSYSTEM:WINDOWS) and click Apply or OK.
and now tryto see if it compiles.
