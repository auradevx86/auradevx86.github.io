so what the hell is the Win32 API even?

the Win32 API is the API that Windows uses to create Windows-specific programs. this is the lowest level of Windows programming that you use. program APIs like Java Swing, wxWidgets and .NET Forms all have calls to the Win32 API! it's just that they handle most of the nitty gritty bits for you, just like higher level programming languages such as Python will have calls to low-level assembly. this is the code that Windows uses directly. this will be what the processor will tell Windows to do. (well, not exactly, because assembly and stuff)


if it's so low level, why should i use it?

warning: lots of tl;dr here, if you don't want to read this, that's fine.


we have stuff like wxWidgets, Qt, GTK+, why use the lowest level?
usually we use the Win32 API for compatibility, flexibility and preference.
lots of the Win32 API have remanents from 16-bit Windows! (i love 16 bit)
but you'll realise that most Win32 applications will run much better than Qt and GTK+.
but it does give a great deal of flexibility. you can easily decide whether you want compatibility for older versions of Windows,
or whether you want the status bar to use this, blah blah blah.
for example, i cannot stress how much i hate using .NET Forms status bar control, for the time i usually spend on trying to get correct a .NET Forms status bar correct I could have created it in the Win32 API, mow the lawn, and hoard 10 kilos of HRT pills. which comes to our final argument, just preference. i've come to realise i don't like using wrapper classes because they usually do all the work for you, and i've always being intrigued about low level stuff. it's the same reason why some people use assembly instead of Python.

the scope of this tutorial

the Win32 API is vast. don't believe me? just go to C:\Program Files (x86)\Windows Kits\ and you'll see everything. that's the Win32 API.

it includes driver interactions, console tools, low level drawing, etc but we're going to focus on the GUI part of everything, because that's what Windows is all about, literal windows.

microsoft has documented the other functions on the low level stuff pretty well, so i don't think i need to go through this.
