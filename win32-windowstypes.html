Win32 API tutorial

part 0: setting up

in the previous page, one of the parameters in wWinMain:

int WINAPI wWinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance, PWSTR pCmdLine, int nShowCmd)

so what is PWSTR?
so Microsoft decided to make things *easier* they added definitions to simply types.
literally, PWSTR is defined like this:

#define PWSTR wchar_t*


usually you can tell what a type is based on this:
  • P is for pointer. (e.g. PSTR = Pointer to string)
  • LP is for long pointer, however in Win32, LP and P is the same thing. (e.g. LPSTR - Long Pointer to String)
  • STR is for string, however it is a character array/pointer to an array instead C++'s string type. (e.g. PSTR)
  • U stands for unsigned. Usually for unsigned integer. (e.g. UINT)
  • W stands for wide. in this case wide for wide character, which is Unicode (UTF-16 in Windows), e.g. (PWSTR = Pointer to Wide String)
  • C stands for const (e.g. LPCSTR = Long Pointer to Constant String).
  • T is just for Windows to decide whether a string should be Unicode or ANSI. (e.g. LPTSTR = Long Pointer to String that is decided by Windows whether you want diversity or not)
  • H is for handle. (e.g. HINSTANCE = Handle to an instance/module)

for example, in this case, PWSTR means Pointer to Wide String.
here's a table of common types you usually use of what these native Windows types means if you can't figure them out:
